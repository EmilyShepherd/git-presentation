What's Version Control?

  • Keeps track of history
  • Assess when code was written
  • ...and in what context it was written
  • Easily revert changes

Why Git?

  • Open Source!
  • Designed to be supper fast
  • Built by the people who use it
  • And most importantly...

Most popular

The Commit

  • Snapshot of the project in time
  • Contain a reference to the previous commit, allowing them to build on top of each other
  • Completely immutable, to make changes either:
    • Make a second commit, or
    • Delete the commit and replace it

What is a Branch?

  • A linear chain of commits
  • A pointer to the commit at the head of the chain

Concurrent Changes

  • Branches can split off from one another:
    • To represent different units of work
    • If two developers make changes on the same branch at the same time
  • To put branches back together, we have two options:
    • Merge, or
    • Rebase
